## 1. 项目概述

本项目以 Ultralytics YOLOv11 为基础框架，针对**太阳能电池板表面缺陷检测**任务进行了两类算法改进：

| 算法 | 定位 | 核心思想 |
|------|------|----------|
| **FAF-YOLOv11n** | 精度优先 | FEM 自适应多尺度卷积 + C3k2_AGA 代理注意力（线性复杂度全局建模） |
| **L-FAF-YOLOv11n** | 边缘端轻量化 | PConv 部分卷积 + EFBlock 高效残差块 + BiFPN 加权特征金字塔 |

检测目标为 5 类缺陷：`black_border`（黑边）、`broken`（破损/裂纹）、`hot_spot`（热斑）、`no_electricity`（断格/无电）、`scratch`（划痕）。

## 3. Ultralytics 框架核心架构

### 3.1 模型加载流程

```
用户代码: YOLO("cfg.yaml")  或  YOLO("weights.pt")
         │
         ▼
    engine/model.py — Model.__init__()
         │
         ├─ .yaml → _new() → yaml_model_load() → DetectionModel(cfg_dict)
         │                        │
         │                        └→ tasks.py / parse_model() — 编译 YAML 为 nn.Sequential
         │
         └─ .pt → _load() → load_checkpoint() → 还原完整模型 + 权重
```

### 3.2 任务分发机制 (task_map)

`Model` 基类通过 `task_map` 属性动态分发到对应实现：

```python
# ultralytics/models/yolo/model.py — YOLO.task_map
{
    "detect":   {"model": DetectionModel,   "trainer": DetectionTrainer, ...},
    "segment":  {"model": SegmentationModel,"trainer": SegmentationTrainer, ...},
    "pose":     {"model": PoseModel,        "trainer": PoseTrainer, ...},
    "classify": {"model": ClassificationModel, ...},
    "obb":      {"model": OBBModel,         ...},
    "semantic": {"model": SemanticSegmentationModel, ...},
}
```

### 3.3 模型解析器 (parse_model) — 核心机制

位于 `ultralytics/nn/tasks.py`，是将 YAML 文本编译为 PyTorch 计算图的**核心路由函数**：

```python
def parse_model(d, ch, verbose=True):
    """
    d  : YAML 解析后的字典 {"backbone": [...], "head": [...]}
    ch : 通道追踪列表 ch[i] 记录第 i 层的输出通道数

    对每一层 [from, repeats, module, args]:
      1. 根据 m (module 类名) 查找对应 PyTorch 类
      2. 从 ch[f] 填入输入通道 c1，根据 args 推导输出通道 c2
      3. 实例化模块，加入 nn.Sequential
      4. 更新 ch 列表
    """
```

### 3.4 训练主干流程

```
BaseTrainer.train()
    │
    ├─ _setup_train()      # 初始化 dataloader, optimizer, scheduler, EMA
    ├─ _do_train()          # Epoch 循环
    │   ├─ preprocess_batch()
    │   ├─ model(batch)     # → 调用 BaseModel.forward()
    │   │   └─ isinstance(x, dict) ? self.loss(x) : self.predict(x)
    │   ├─ loss.backward()
    │   └─ optimizer.step()
    ├─ validate()           # 每个 val_interval epoch
    └─ save_model()         # 保存 best.pt / last.pt
```

## 4. 两套改进算法详解

### 4.1 FAF-YOLOv11n — 精度优先架构

**改进策略**：增强骨干网络对小缺陷的特征抓取能力

| 模块 | 作用 | 创新点 |
|------|------|--------|
| **FEM** | 自适应多尺度特征提取 | 三路分流卷积 (不同扩张率) + 通道级动态权重融合 |
| **C3k2_AGA** | 高效全局注意力 | 代理注意力 (Agent Attention) — O(NMd) 线性复杂度 |
| **C2PSA** | 跨阶段部分自注意力 | 在 SPPF 后引入自注意力增强顶层特征 |

**数据流路径**：

```
Input → Conv → C3k2_AGA → Conv → C3k2_AGA → FEM → C3k2_AGA → Conv → C3k2_AGA → SPPF
                                                                                      ↓
       Detect ← C3k2_AGA ← Concat ← C3k2 ← Concat ← C3k2_AGA ← Concat ← C3k2_AGA ← Neck
```

### 4.2 L-FAF-YOLOv11n — 边缘端轻量化架构

**改进策略**：大幅削减计算量，适配边缘设备实时推理

| 模块 | 作用 | 创新点 |
|------|------|--------|
| **PConv** | 低 MAC 下采样 | 仅对 1/4 通道做空间卷积，其余恒等映射 |
| **EFBlock** | 高效残差块 | 分组卷积 + ELA 注意力 + 残差直连 |
| **BiFPN_Concat** | 加权特征融合 | 可学习权重自动调节多尺度特征贡献 |
| **C2PSA** | 顶层自注意力 | 同 FAF |

**数据流路径**：

```
Input → PConv → PConv → EFBlock → PConv → EFBlock → PConv → EFBlock → PConv → EFBlock → SPPF → C2PSA
                                                                                                  ↓
       Detect ← EFBlock ← BiFPN ← EFBlock ← BiFPN ← EFBlock ← BiFPN ← Upsample ←────────────→ Neck
```

### 4.3 性能对比

| 指标 | YOLOv11n (Baseline) | FAF-YOLOv11n | L-FAF-YOLOv11n |
|------|---------------------|--------------|----------------|
| 参数量 | 3.12M | 3.88M | **2.89M** |
| 计算量 | 6.5 GFLOPs | 6.2 GFLOPs | **4.8 GFLOPs** |
| mAP@0.5 提升 | 基准 | **+6.6%** | 小幅提升 |
| T4 TensorRT | 1.5ms | 2.9ms | 1.8ms |

## 5. 自定义模块源码剖析

所有自定义模块位于 `ultralytics/nn/modules/block.py`，以下按功能分组说明。

### 5.1 高效注意力系列

**ELA** (Efficient Local Attention) — `block.py:2086`
```
核心公式: y = x * σ(GN(Conv1d(AvgPool_H(x)))) * σ(GN(Conv1d(AvgPool_W(x))))
```
- 无降维条带池化（保留精确坐标信息）
- 1D 分组卷积实现轻量级通道交互
- 适用场景：替代 SE/CBAM，为 EFBlock 提供空间注意力

**AgentAttention** — `block.py:2152`
```
核心思想: 引入 M 个代理 Token (M << N), 将 O(N²d) 降为 O(NMd)
步骤1: V_A = Softmax(AK^T/√d) V       # 代理聚合全局信息
步骤2: Out = Softmax(QA^T/√d) V_A     # 广播回每个位置
```
- 线性复杂度全局注意力
- C3k2_AGA 的核心组件

### 5.2 高效卷积系列

**PConv** (Partial Convolution) — `block.py:2123`
```
操作: Y = Concat(Conv3×3(X[:, :Cp]), X[:, Cp:])
```
- 仅对前 1/4 通道做空间卷积
- 利用特征图通道冗余，降低 MAC

**FEM** (自适应多尺度特征处理卷积) — `block.py:2235`
```
三路并行: 上(d=1) + 中(d=2) + 下(d=4)
自适应融合: F_fused = α*F_up + β*F_mid + γ*F_low
权重由 SE-like 模块动态预测 (α+β+γ=1)
```

**EFBlock** (Efficient Fusion Block) — `block.py:2310`
```
结构: Input → PConv (通道扩展 e=0.75) → ELA 注意力 → PConv (投影回原通道) + Residual
```
- L-FAF-YOLOv11n 的主力残差块
- 当输入/输出通道不同时用 1×1 卷积做投影捷径

### 5.3 特征融合系列

**BiFPN_Concat** — `block.py:2276`
```
公式: O = Σ(w_i / (ε + Σw_j)) * I_i
```
- 可学习权重 w_i 经 ReLU 激活保证非负
- 支持 2/3 路输入加权融合
- Fast Normalized Fusion 策略

**C3k2_AGA** — `block.py:2217`
- 继承自 C2f，在其 Bottleneck 中嵌入 AgentBottleneck
- AgentBottleneck = 标准卷积 + AgentAttention 支路

### 5.4 其他自定义模块

| 模块 | 位置 | 功能 |
|------|------|------|
| **A2C2f** | block.py:1802 | 带注意力的 C2f 变体 |
| **ADown** | block.py:947 | 带注意力的下采样 |
| **AConv** | block.py:928 | 带注意力的卷积 |
| **SCDown** | block.py:1537 | 空间到通道下采样 |
| **C2PSA** | block.py:1443 | 跨阶段部分自注意力 |
| **CBAM** | conv.py | 通道+空间注意力 |
| **SwiGLUFFN** | block.py:1884 | SwiGLU 前馈网络 |
| **SAVPE** | block.py:1928 | 自注意力视觉位置编码 |

## 6. 模型配置 YAML 解读

### 6.1 L-FAF-YOLOv11n 配置 (`l_faf_yolov11n.yaml`)

```yaml
nc: 5              # 类别数: black_border, broken, hot_spot, no_electricity, scratch
scales:
  n: [0.50, 0.25, 1024]  # [depth_multiple, width_multiple, max_channels]

backbone:
  # 格式: [from, repeats, module, args]
  - [-1, 1, Conv, [64, 3, 2]]       # 0: 标准卷积下采样 (H/2, W/2)
  - [-1, 1, Conv, [128, 3, 2]]      # 1: 标准卷积下采样 (H/4)
  - [-1, 1, EFBlock, [128, True]]   # 2: 高效残差块 (含 ELA)
  - [-1, 1, PConv, [256, 3, 2]]     # 3: 部分卷积下采样 (H/8)
  - [-1, 1, EFBlock, [256, True]]   # 4: 高效残差块
  - [-1, 1, PConv, [512, 3, 2]]     # 5: 部分卷积下采样 (H/16)
  - [-1, 1, EFBlock, [512, True]]   # 6: 高效残差块
  - [-1, 1, PConv, [1024, 3, 2]]    # 7: 部分卷积下采样 (H/32)
  - [-1, 1, EFBlock, [1024, True]]  # 8: 高效残差块
  - [-1, 1, SPPF, [1024, 5]]        # 9: 空间金字塔池化
  - [-1, 2, C2PSA, [1024]]          # 10: 跨阶段自注意力

head:
  # 自顶向下 + BiFPN 加权融合
  - [-1, 1, nn.Upsample, [None, 2, "nearest"]]         # 11: 上采样
  - [[-1, 6], 1, BiFPN_Concat, []]                     # 12: 加权融合 (P5↑ + P4)
  - [-1, 1, EFBlock, [512, True]]                       # 13: 特征精炼

  - [-1, 1, nn.Upsample, [None, 2, "nearest"]]         # 14: 上采样
  - [[-1, 4], 1, BiFPN_Concat, []]                     # 15: 加权融合 (P4↑ + P3)
  - [-1, 1, EFBlock, [256, True]]                       # 16: P3 检测输入

  # 自底向上路径
  - [-1, 1, PConv, [256, 3, 2]]                        # 17: 下采样
  - [[-1, 13, 6], 1, BiFPN_Concat, []]                 # 18: 三路加权融合
  - [-1, 1, EFBlock, [512, True]]                       # 19: P4 检测输入

  - [-1, 1, PConv, [512, 3, 2]]                        # 20: 下采样
  - [[-1, 10], 1, BiFPN_Concat, []]                    # 21: 加权融合
  - [-1, 1, EFBlock, [1024, True]]                      # 22: P5 检测输入

  - [[16, 19, 22], 1, Detect, [nc]]                    # 23: 多尺度解耦检测头
```

### 6.2 YAML 配置语法说明

| 字段 | 含义 | 示例 |
|------|------|------|
| `from` | 输入来源层索引, `-1`=上一层, `[a,b]`=拼接多层 | `[[-1, 6]]` |
| `repeats` | 模块重复次数, 实际值 = repeats × depth_multiple | `3` |
| `module` | PyTorch 类名或内置模块字符串 | `Conv`, `C3k2_AGA`, `nn.Upsample` |
| `args` | 模块初始化参数 | `[64, 3, 2]` = [out_ch, kernel, stride] |

## 9. 开发规范与模块注册流程

> 本项目通过 `CLAUDE.md` 定义了严格的开发规约，以下为自定义模块的完整注册流程。

### 9.1 三步注册流程

```
Step 1: 在 ultralytics/nn/modules/block.py 中实现 PyTorch 模块
         ↓
Step 2: 在 ultralytics/nn/modules/__init__.py 中导入并加入 __all__
         ↓
Step 3: 在 ultralytics/nn/tasks.py 的 parse_model() 中添加通道路由逻辑
```

### 9.2 Step 1 — 模块实现

```python
# ultralytics/nn/modules/block.py

class MyCustomModule(nn.Module):
    """自定义模块的数学公式与功能描述。"""
    def __init__(self, c1, c2, k=3):
        super().__init__()
        self.conv = nn.Conv2d(c1, c2, k, padding=k//2)

    def forward(self, x):
        return self.conv(x)
```

### 9.3 Step 2 — 接口暴露

```python
# ultralytics/nn/modules/__init__.py

from .block import MyCustomModule  # 添加导入

__all__ = (
    # ... 已有模块 ...
    "MyCustomModule",              # 添加到 __all__
)
```

### 9.4 Step 3 — 解析路由

```python

            c2 = args[0]

```

### 9.5 关键原则

- **内存优先**：注意力路径中避免密集 2D 卷积
- **PEP 8**：严格遵循 Python 代码风格
- **数学注释**：数学密集的变换需保留详细注释
- **统一管理**：不要在 `block.py` 之外创建独立模块脚本
